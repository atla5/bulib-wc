# Grid Layout

Grid layouts aligns things in a specified number of columns (determined by `grid-template-layout`).
  The number of columns is the same regardless of the width of the page, and the content within 
  them stretches to fit. 

If unspecified, `.grid` defaults to a single column, which is normally not what you want:

## Specifying the Number of columns

To specify the number of columns in our helpers, use `.grid-2`, `.grid-3`, or `.grid-4`.

## Responsive

To make the base `grid` layout more responsive, try decreasing the number of columns you're using
  for smaller width using `grid-template-columns` and `@media` queries in your custom css:

```css
/* reduce the number of columns for a particular grid section for small screens */
@media only screen and (max-width: 850px){
  div.grid-4.grid-responsive { grid-template-columns: auto auto; }
}
```

## Advanced Layout 

You can do [quite a lot](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Grid_Layout) 
  with CSS Grid beyond just specifying columns, and even layout whole pages and complex 
  UIs whose contents shift at various widths.

See the [`bulib-footer` web component](?path=/story/branding-bulib-footer--default-story) 
  for an example of this.
